## 1. 项目概述

### 1.1 产品定位
**ImageMatte**（暂定）是一款基于AI的在线图片背景去除工具，专注于为电商、设计、自媒体等场景提供快速、精准的人像抠图服务。

### 1.2 核心价值主张
- **极速处理**: 3秒内完成高质量背景去除
- **零门槛**: 无需PS技能，一键操作
- **专业效果**: AI算法优化，边缘处理精准
- **多场景适配**: 电商产品图、人像摄影、设计素材

### 1.3 目标用户画像
| 用户类型 | 核心需求 | 使用频率 | 付费意愿 |
|---------|---------|---------|---------|
| **电商卖家** | 商品图背景统一、白底图制作 | 高频（每周） | 中等 |
| **设计师** | 快速获取透明素材、合成素材 | 中频 | 较高 |
| **自媒体从业者** | 封面图制作、内容配图处理 | 高频 | 中等 |
| **个人用户** | 证件照处理、日常照片美化 | 低频 | 较低 |

### 1.4 市场竞争分析
| 竞品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| Remove.bg | 市场领导者、技术成熟 | 免费额度少、价格较高 | 更友好的中文界面、更快的处理速度 |
| ClipDrop | 功能丰富、多工具集成 | 界面复杂、学习成本高 | 专注单一功能、极致简化 |
| Photopea | 功能全面、免费 | 专业性强、操作复杂 | 零门槛、快速出结果 |

## 2. MVP功能规格

### 2.1 核心功能模块

#### 2.1.1 图片上传模块
- **支持格式**: JPG、PNG、WebP（覆盖99%用户需求）
- **文件限制**:
  - 大小: ≤10MB（平衡用户体验与处理性能）
  - 分辨率: ≤4K（4096×4096，支持高清需求）
  - 宽高比: 1:10 ~ 10:1（防止异常图片）
- **上传方式**:
  - 拖拽上传（桌面端优化）
  - 点击选择文件（全平台兼容）
  - 粘贴上传（Ctrl+V，设计师高频需求）
  - 移动端相册选择（响应式适配）
- **实时验证**:
  - 格式验证（前端MIME类型检测）
  - 大小验证（即时提示）
  - 尺寸验证（防止超限图片）

#### 2.1.2 背景去除处理模块
- **处理流程**:
  ```
  用户上传 → 前端Base64编码 → API代理 → Remove.bg API → 返回透明PNG → 前端解码展示
  ```
- **性能指标**:
  - 平均处理时间: <3秒（90%场景）
  - 成功率: >98%（网络正常情况）
  - 超时机制: 15秒自动取消
- **API配置**:
  - 服务商: Remove.bg（技术成熟、效果稳定）
  - 免费额度: 50张/月（MVP测试足够）
  - 请求参数: `size=auto`, `type=person`（人像优化）

#### 2.1.3 结果预览与编辑模块
- **预览模式**:
  - 左右对比视图（原图 vs 结果）
  - 独立视图切换
  - 缩放控制（25%~400%）
  - 平移查看细节
- **背景切换**:
  - 纯色背景: 白、黑、透明、自定义颜色
  - 网格背景: 检查边缘处理质量
  - 图片背景: 简单合成预览（后续版本）
- **基础编辑**:
  - 图片裁剪（保持比例/自由裁剪）
  - 尺寸调整（预设电商尺寸）
  - 格式转换（PNG/JPG/WebP）

#### 2.1.4 下载与分享模块
- **下载选项**:
  - 格式: PNG（透明背景）、JPG（白色背景）
  - 质量: 高（90%）、中（75%）、低（50%）
  - 尺寸: 原尺寸、预设电商尺寸（800×800等）
  - 命名: `ImageMatte_YYYYMMDD_HHMMSS.png`
- **分享功能**:
  - 生成临时分享链接（24小时有效）
  - 社交媒体分享（微信、微博）
  - 复制图片到剪贴板

### 2.3 错误处理与边界情况

#### 2.3.1 错误类型与处理
| 错误类型 | 触发条件 | 用户提示 | 处理方式 |
|---------|---------|---------|---------|
| 格式错误 | 不支持的文件类型 | "请上传JPG、PNG或WebP格式图片" | 阻止上传，明确提示 |
| 大小超限 | >10MB | "图片大小不能超过10MB" | 阻止上传，建议压缩 |
| 尺寸超限 | >4K分辨率 | "图片分辨率不能超过4096×4096" | 阻止上传，建议调整 |
| 网络错误 | API调用失败 | "网络连接不稳定，请重试" | 自动重试1次，提供手动重试 |
| API限额 | 免费额度用完 | "本月免费额度已用完" | 提示升级方案，提供等待选项 |
| 处理失败 | AI识别失败 | "背景去除失败，请尝试其他图片" | 提供帮助文档链接 |

#### 2.3.2 加载状态设计
- **上传中**: 进度条 + 取消按钮
- **处理中**: 动画效果 + 预估时间
- **网络慢**: 超时提示 + 重试建议
- **成功完成**: 动效反馈 + 操作引导

## 3. 技术架构设计

### 3.1 系统架构图
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   用户浏览器    │    │ Cloudflare Pages│    │  Remove.bg API  │

├─────────────────┤    ├─────────────────┤    ├─────────────────┤
│ 1. 图片上传     │───▶│ 2. Base64编码   │───▶│ 3. AI处理       │
│ 5. 结果展示     │◀───│ 4. 结果返回     │◀───│    (3秒)        │
│ 6. 下载/分享    │    │                 │    │                 │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                        │
         └────────────────────────┘
           API代理 (pages/api/process.js)
```

### 3.2 前端技术栈
- **框架**: Next.js 15（App Router，服务端组件）
- **样式**: Tailwind CSS + CSS Modules
- **状态管理**: React Context + useReducer
- **文件处理**: File API + Canvas API
- **HTTP客户端**: axios（错误拦截、重试机制）
- **代码质量**: ESLint + Prettier + TypeScript

### 3.3 API代理设计
```typescript
// pages/api/process.ts
interface ProcessRequest {
  image: string; // Base64编码
  options?: {
    size?: 'auto' | 'preview' | 'full';
    type?: 'auto' | 'person' | 'product';
  };
}

interface ProcessResponse {
  success: boolean;
  data?: string; // Base64结果图片
  error?: {
    code: string;
    message: string;
    details?: any;
  };
  metadata?: {
    processingTime: number;
    originalSize: number;
    resultSize: number;
  };
}
```

### 4.2 安全性要求
1. **API密钥保护**
   - 存储: Cloudflare Secrets（非代码仓库）
   - 访问: 仅API代理层可访问
   - 轮换: 每月自动轮换（后续）

2. **请求限制**
   - 频率限制: 10次/分钟/IP（防止滥用）
   - 文件验证: 前端+后端双重验证
   - 大小限制: 严格10MB上限

3. **数据隐私**
   - 无持久化存储: 图片仅内存处理
   - 无用户追踪: MVP阶段不收集个人信息
   - 透明政策: 明确说明数据处理方式

### 4.3 可访问性要求
- **WCAG 2.1 AA标准**: 基础合规
- **键盘导航**: 完整Tab顺序，操作提示
- **屏幕阅读器**: ARIA标签，语义化HTML
- **颜色对比度**: 4.5:1最低标准
- **字体大小**: 支持浏览器缩放
